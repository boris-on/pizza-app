Правила работы:
1. Совместно создаются задачи на реализацию необходимого функционала приложения, в ходе работы каждый член команды
в праве создать задачу на исправление бага или доработку фичи
2. Член команды берёт в работу задачу, назначая себя исполнителем, создаёт отдельно ветку под свою задачу с одноименным названием.
3. После выполнения всех требований задачи человек отправляет merge request в git
4. Далее необходимо одобрение минимум 2 членов команды на ревью (50% команды), Для того чтобы задачу смержили с основной веткой dev
5. После одобрения командой система git сама соединит ветку с задачей с основной веткой. В случае, когда есть конфликты, разработчик их решит непосредственно после одобрения на ревью.

Доп. информация:
1. Все задачи описаны непосредственно в git проекте, решили обойтись без jira, по хорошему всё очень похожее.
2. Любой член команды в праве создавать задачу, если есть спорные моменты, то член команды обсуждает идею и только после этого добавляет задачу.
3. Ветки с версиями также хранятся в гит репозитории, ведётся версионность у приложения.
